Fraunces 700 · Inter 400/500/600 · Mono 500

Rationale

Concept. ePepper as a kitchen-shelf magazine — print-borrowed typographic hierarchy, generous margins, hairline rules, small-caps section labels, a single warm paprika accent that means something when it appears.

Mood. Quiet, confident, a little austere. Closer to Cherry Bombe or a Phaidon cookbook than a SaaS dashboard.

Type. Fraunces for display headings and recipe titles (gentle SOFT axis, italic 'e' as brand mark). Inter for UI/body. JetBrains Mono only for the access-code chip — the deliberate sans/serif contrast is the design.

Color. Bone-white paper, deep ink, single paprika red (#B83227 light / #E5614F dark) for primary action and the on-display badge. No green: paprika pulls toward cooking, the original green pulled toward wellness.

Layout. Wider 1100px canvas with a 680px reading column; recipe pages are an editorial 2-column grid (sticky ingredients sidebar, instructions with oversized italic step numerals 01–06). Library is a vertical index — no cards, rule dividers, serif titles, tracked small-caps source labels.

Rationale

Concept. A private recipe library that feels personal, not enterprise. Late-morning light through a window, a saffron tea towel, a stub of pencil in the margin — playful, warm, hand-crafted; never twee.

Type. Fraunces (variable, opsz + SOFT) for display — quirky modern serif, almost hand-lettered at huge sizes, sharp at H2. Inter for body. JetBrains Mono for the access-code chip.

Color. Low-saturation surfaces, saturated accents. Warm parchment #FBF4E4 / inky raisin #1F1A1D. Saffron #E8A33D as hero (brand, primary action, link underlines); mineral teal #2C6E6A supports; persimmon #B23A1F for danger.

Polish moves. Soft 14px radii, a tape-strip on the login card, a pencilled dashed underline beneath the topbar, saffron strip down each library card, numbered ingredient bullets, circular saffron tokens for step numbers, a slightly-rotated polaroid frame for the e-ink preview, stamp-style "on display" pill.

Layout. Recipe pages get a two-column reading layout on desktop (ingredients in a sticky aside on a warm sub-panel); collapses to single column on mobile. Status page wraps the BMP preview in a polaroid frame.

Space Grotesk 500/700 · Inter · JetBrains Mono labels

Rationale

Concept. A kitchen tool, not a magazine. A self-hosted library that broadcasts to an e-ink screen sitting on a counter near steam and grease — structural, legible across the room, indifferent to fashion.

Mood. Raw, opinionated, high-contrast. Hairline rules instead of shadows. Boxes that read as boxes. Mono labels are the brutalist tell — counts, sources, indices read like file metadata.

Color. Bone-paper light, graphite dark. One signal color: hot vermillion (#E33B1B / #FF5A3C) for primary action and the on-display indicator. Danger is outline-only — destruction never looks inviting.

Layout. 12-column grid, 1180px max. Recipe pages are an asymmetric two-column split (sticky ingredients aside + wider reading column with mono accent step numbers). Library is a numbered directory listing — 01 / TITLE / SOURCE / META. Topbar shows a mono route /RECIPES. Sections get a vermillion vertical bar.

Brutalism, not brutality. Sharp edges, no shadows, no gradients — but generous line-height and a single accent color keep it readable rather than aggressive.

21.4 °C · 47 % humidity · -58 dBm

Fraunces display · Nunito body · Mono metrics

Rationale

Concept. ePepper lives on a kitchen counter next to a stack of cookbooks and a pot of basil. Rounded edges, herb-garden palette, hand-pulled feel without sacrificing legibility — a sun-bleached recipe card pinned to a cork board.

Mood. Calm, unhurried, generous with whitespace. Pillowy shapes (16–28px corner radii everywhere, including inside forms and buttons), diffused shadows, warm low-contrast borders. The e-ink preview gets a soft frame.

Type. Fraunces for display (soft humanist serif with optical sizing — perfect across English/French/Italian recipes). Nunito for UI/body (high x-height, rounded terminals, excellent at small sizes in dark mode). JetBrains Mono only for metrics so -58 dBm doesn't get mistaken for prose.

Color. Oat paper #F6F1E6 with moss-green primary and terracotta secondary; forest-floor #1C211E dark lifted by sage #A8C89A. Every accent has a paired soft tint.

Layout. Recipe pages: ingredients pinned in a sage sidebar, instructions flowing beside; collapses on mobile. Library cards have a moss vertical accent stripe that brightens on hover. Status page treats each section as a rounded tile with a tinted icon medallion.

Space Grotesk display · Inter body · Mono metrics

Rationale

Concept. ePepper as self-hosted infrastructure: library = database, e-ink display = device, LLM = service. A small terminal-adjacent control panel — dense, precise, monospaced where it counts, generous in recipe reading. Linear's command palette meets a Raspberry Pi admin page meets a well-typeset recipe card.

Mood. Precise (8px grid, 4px corner radius, no shadows except hairlines), high contrast, dense but breathable.

Type. Space Grotesk for display, Inter for body, JetBrains Mono for nav labels, badges, metrics, ID-like data (COOKED 7×, LAST 4D, BATT 76%). The mono establishes the developer-tool register.

Color. Two themes: warm off-white / deep cool neutral. Single saturated pepper-green accent (#4F8A1F / #A8E861) for brand, primary CTA, focus state, on-display badge.

Layout. Sticky topbar with mono route after the brand (ePepper / library ?q=pasta, ePepper / recipes #01). Library is a flat dense table with an ID column (#01, #02…). Recipe pages: 280px sidebar + main column with mono step numbers 01, 02… and h2 data-label captions like 8 items, 6 steps. Status page is a stat-tile grid with /device/screen floating labels.
